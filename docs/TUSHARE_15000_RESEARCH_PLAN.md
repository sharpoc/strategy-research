# Tushare 15000 积分研究计划

最后更新：`2026-04-10`

## 当前定位

这份文档用于给 `Mac mini` 接手研究时使用。

用户已经拿到 `Tushare 15000` 积分级别 token。这里不记录 token 本身，只记录：

- 15000 积分最值得利用的接口方向
- 哪些能力仍不属于积分权限
- 现有策略应该怎么升级
- `Mac mini` 下一步应该怎么验证

## 权限边界

根据当前 Tushare 官方文档口径，15000 积分最有价值的点不是实时盘中，而是特色数据和筹码类数据的使用空间更大。

当前优先关注：

- `cyq_perf`
  - 每日筹码及胜率
  - 适合做筹码获利比例、成本区、胜率变化、筹码改善确认
- `cyq_chips`
  - 每日筹码分布
  - 适合判断上方套牢压力、下方成本支撑、低位筹码集中
- `stk_factor_pro`
  - 技术面因子专业版
  - 适合统一提供 `MA / MACD / KDJ / 趋势状态` 等技术指标底座

需要注意：

- 15000 积分不等于自动拥有实时行情权限
- `rt_min`、历史分钟、实时分钟、盘前实时等仍应按 Tushare 官方权限单独确认
- 因此这份计划先聚焦“日线级别策略增强”，不把盘中实时作为第一阶段目标

## 总体策略方向

15000 积分最适合把现有体系升级成：

`事件/资金/趋势初筛 -> 筹码确认 -> 技术结构确认 -> 只保留高质量信号`

当前不建议一上来新增很多完全独立策略。更稳的路径是：

1. 先强化现有最接近线上主线的 `星曜增持臻选`
2. 再强化平台/突破类策略的筹码确认层
3. 最后再尝试新的“低位筹码集中 + 趋势初修复”策略

## 优先方向一：星曜增持臻选筹码强化版

### 目标

把 `星曜增持臻选` 从：

`资金 + 趋势 + 业绩 + 风控 + 增持辅助`

升级成：

`资金 + 趋势 + 业绩 + 风控 + 增持辅助 + 筹码确认`

### 建议使用接口

- `cyq_perf`
- `cyq_chips`
- `stk_factor_pro`

### 核心改造点

只改研究层，先不上线上：

- 用 `cyq_perf` 提供：
  - `winner_rate`
  - `winner_rate_change_5d`
  - 当前价相对筹码成本区的位置
- 用 `cyq_chips` 判断：
  - 上方套牢压力是否重
  - 当前价下方是否有密集成本支撑
  - 价格突破后是否仍面对明显筹码峰
- 用 `stk_factor_pro` 统一技术指标：
  - `MA5 / MA10 / MA20 / MA60 / MA250`
  - `MACD`
  - 量比或技术状态字段

### 研究假设

如果候选票具备以下特征，应加分：

- `winner_rate` 持续改善
- 当前价仍未严重脱离主要成本区
- 上方筹码压力不重
- 筹码峰在当前价下方形成支撑
- 资金和趋势同步改善

如果出现以下特征，应降权：

- 上方套牢区很重
- 当前价远高于主要成本区
- `winner_rate` 已经过热
- 股价涨了但筹码结构没有改善

## 优先方向二：平台突破/回踩策略的筹码确认版

适用策略：

- `天衡回踩转强臻选`
- `真实资金突破臻选`

### 目标

过滤“看起来突破，但上方全是套牢盘”的假突破。

### 建议使用接口

- `cyq_chips`
- `cyq_perf`
- `stk_factor_pro`

### 核心改造点

在平台突破或回踩转强之后，增加筹码确认：

- 当前价附近是否有成本支撑
- 平台上沿上方是否存在明显高筹码压力
- 突破时 `winner_rate` 是否健康提升
- 回踩时是否仍守住主要成本区

### 研究假设

更优信号通常是：

- 平台整理阶段筹码收敛
- 突破前上方压力不重
- 突破后获利盘没有极端过热
- 回踩缩量且不跌破主成本区

## 优先方向三：低位筹码集中 + 趋势初修复

### 目标

新建一条研究策略，不急着上线。

寻找：

- 价格处于中低位
- 筹码开始集中
- 上方压力不重
- `MA10 / MA20` 开始修复
- 资金不再持续流出

### 建议使用接口

- `cyq_perf`
- `cyq_chips`
- `stk_factor_pro`
- `moneyflow`
- `daily_basic`

### 核心信号

优先观察：

- 当前价处于 `120/250` 日区间中低位
- `winner_rate` 从低位回升
- 主成本区接近当前价
- 上方高成本筹码带较轻
- `MA10` 走平或向上
- 收盘重新站上 `MA20`

## Mac mini 第一阶段执行清单

### 1. 先验证接口稳定性

在 `Mac mini` 上先做小范围接口探测，不要一上来全市场扫：

- 取 `10` 只股票
- 取最近 `5` 个交易日
- 分别验证：
  - `cyq_perf`
  - `cyq_chips`
  - `stk_factor_pro`

输出目标：

- 每个接口是否有权限
- 每分钟稳定吞吐大概是多少
- 是否有字段缺失
- 是否仍会出现 `Read timed out / RemoteDisconnected`

### 2. 再做缓存层

建议先做本地缓存，不直接把接口调用写进策略评分主流程：

- `output/cache/tushare/cyq_perf/`
- `output/cache/tushare/cyq_chips/`
- `output/cache/tushare/stk_factor_pro/`

缓存原则：

- 按 `trade_date` 分文件
- 已存在则不重复拉
- 出错时记录缺口，不整批失败
- 不提交到 Git

### 3. 再做策略 A/B

优先做：

`星曜增持臻选 baseline` vs `星曜增持臻选 + 筹码确认`

核心指标：

- `3/5/10` 日收益
- 胜率
- 最大回撤
- 样本数
- 是否明显减少高位假信号

### 4. 再考虑是否推到展示系统

只有当本地研究证明明显提升后，再考虑：

- 是否接入 `strategy-lab`
- 是否让 `Mac mini` 推送结果到线上
- 是否保留旧版作为对照

## 当前不要做的事

- 不要把 token 写进仓库
- 不要把行情、筹码、回测结果提交进 Git
- 不要直接在 ECS 上跑这些重任务
- 不要先做盘中实时策略
- 不要把 `cyq_chips` 全市场全历史一次性拉爆接口

## 建议下一步

Mac mini 接手后，下一步最推荐：

1. 写一个 `probe_tushare_15000_data.py`
2. 小样本验证 `cyq_perf / cyq_chips / stk_factor_pro`
3. 写本地缓存层
4. 在 `星曜增持臻选` 上做 A/B

如果这些接口稳定，再继续做平台突破筹码版和低位筹码集中策略。
